CPUFlow Architecture

Designed from CPU hardware principles — not adapted from GPU designs. Every component exploits CPU advantages.

core mixing layer
x_n = PowerNorm(x) c = tanh(W_v(x_n)) # compress d→k g = sigmoid(W_g(x_n)) # learned gate s = cumsum(c ⊙ g) # sequential scan s = W_m(PowerNorm(s)) # self-mix x = x + W_e(s) # expand + residual h = GELU(ff_up(PN(x))) # per-position FFN x = x + ff_down(h)

Key Findings

What we learned from 30+ experiments and 21 failed attempts.

01

Objective > Architecture

Future Sentence Prediction improved perplexity 2.5× with only 1.7% parameter overhead. All 21 architecture-only changes failed to match this gain.

FSP: 10.24 PPL vs baseline: 25.08 PPL
02

Sequential > Parallel (on CPU)

CumMix replaces O(n²) attention with O(n) cumulative sum. The CPU prefetcher handles sequential scans natively, avoiding the memory bottleneck of attention.

89 μs vs 2,062 μs per layer
03

Stripping ≠ Designing

CPUFlow v1 stripped all GPU components and got PPL 329. Removing GPU ops without adding CPU-native learning capacity (normalization, FFN) made the model worse, not better.

v1: PPL 329 → v2: PPL 25.2
04

Scale Is the Ceiling

Our only coherent model required 29.7M params and 40 hours. Below ~10M, models learn vocabulary and syntax but lack narrative state capacity for coherent stories.

PPL 1.36 @ 29.7M params
