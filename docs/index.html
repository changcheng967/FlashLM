Honest Assessment

v9.7 achieves semi-coherent generation at PPL 10.23. Stories have named characters, pronoun tracking, dialogue, and narrative structure. But coherence breaks down ~100 tokens in with word salad and contradictions.

PPL ≠ coherence. v8 (PPL 9.30, hard slot routing) is incoherent. v5-LN (PPL 11.94, cumsum only) is semi-coherent. v9.7 (PPL 10.23, cumsum + memory) is also semi-coherent. The cumsum backbone drives coherence, not PPL or entity routing.

Entity tracking remains unsolved at 2.5M parameters. Six different mechanisms failed: softmax memory (v7), discrete slots (v8), supervised slots (v8.5), Product Softmax (v9), contrastive routing (v9.5), two-phase contrastive (v9.6). The binding threshold (~160M params) is a fundamental barrier.

The key positive finding: sparse memory adds capacity without breaking coherence. RAM-Net's Product Softmax (512 slots, Top-8) improves PPL by 1.7 points as a parameter-efficient capacity expansion. This is useful for CPU-native models.

CPUFlow Evolution

From minimal scan to memory-enhanced coherent generation. Each version isolates one change.

CPUFlow v1 — Minimal Scan

compress→relu→gate→cumsum→expand. No norms, no position encoding, no FFN. Proved cumsum works but PPL 260.

1.34M params PPL 260 11K tok/s

CPUFlow v2 — Stability

Added PowerNorm, CumStepPos, FFN, FSP. NaN at step 1036 revealed that normalization choice matters.

~2M params PPL 25.2 5.7K tok/s

CPUFlow v3 — Linear Attention Cumsum

q·cumsum(kv)/cumsum(k). First stable CPU-native model. Bidirectional v4 failed (leaks future).

1.99M params PPL 25.00 6.1K tok/s

CPUFlow v5-PN — Fused Projection + PowerNorm

Fused Q/K/V (one matmul instead of three). NaN at step 1020. PowerNorm was 57% of compute.

2.0M params PPL 28.59 6.1K tok/s

CPUFlow v5-LN — LayerNorm Breakthrough

PowerNorm → LayerNorm (MKL fused kernel, 27x faster). Zero NaN over 6,886 steps. Coherent baseline at PPL 11.94.

2.0M params PPL 11.94 7.8K tok/s 0 NaN

CPUFlow v7 — Soft Memory Bank (failed)

32-slot content-addressable memory. Cold start: gates collapsed (stayed at 0.12). Warm start: gates opened but softmax still blended across slots.

2.26M params PPL 13.72 (warm) 5.8K tok/s

CPUFlow v8 — Discrete State Streams

Hard argmax routing + straight-through estimator. Best PPL (9.30) but incoherent — discrete routing disrupts continuous context.

2.0M params PPL 9.30 7K tok/s incoherent

CPUFlow v9 — RAM-Net Sparse Memory

Product Softmax addressing (3×8 = 512 virtual slots, Top-8). Low PPL (9.67) but still incoherent — entity addresses collapsed.

2.57M params PPL 9.67 3.6K tok/s incoherent

CPUFlow v9.7 — Memory-Enhanced Coherent Model

v5-LN cumsum backbone + RAM-Net memory sidepath (no entity routing). Coherent generation with 1.7 PPL improvement over baseline.

2.47M params PPL 10.23 3.4K tok/s coherent

Key Findings

What 40+ experiments taught us about CPU-native language modeling.

01

Loss > Architecture

Future Sentence Prediction improved PPL 2.5× with 1.7% parameter overhead. All 21 architecture-only changes failed to match this gain.

FSP: 10.24 PPL vs baseline: 25.08 PPL
02

PPL ≠ Coherence

v8 (PPL 9.30, hard slot routing) produces incoherent text. v5-LN (PPL 11.94, cumsum only) is semi-coherent. v9.7 (PPL 10.23, cumsum + memory) is also semi-coherent. The cumsum backbone drives coherence, not PPL or discrete routing.

v8 PPL 9.30 incoherent, v9.7 PPL 10.23 coherent
03

Operation Speed > Cleverness

PowerNorm (learned exponent) was 57% of compute. LayerNorm (MKL fused kernel) is 27x faster. More steps per second beats better per-step learning.

v5-PN: NaN at step 1020 vs v5-LN: 0 NaN in 6,886 steps
04

C++ Kernels < PyTorch MKL

Custom AVX2 fused scan kernel ran at 0.54x speed of pure PyTorch. MKL-optimized matmuls and vectorized ops are hard to beat with hand-written kernels.

Custom C++: 0.54x PyTorch speed
05

FFN Is Load-Bearing

Removing FFN saves 18% compute but doubles PPL (11.94→22.75). The per-position nonlinear transformation is essential.

NoFFN: PPL 22.75 vs full: PPL 11.94
06

Entity Tracking Fails at 2.5M Params

Six mechanisms tried: softmax memory (v7), discrete slots (v8), supervised slots (v8.5), Product Softmax (v9), contrastive routing (v9.5), two-phase contrastive (v9.6). All collapsed to mode-averaging. The binding threshold (~160M params) is real.

All 6 entity routing attempts collapsed
07

97% Dispatch Overhead

On CPU, PyTorch dispatch overhead dominates over actual compute. CPUFlow minimizes operations from 233 to ~50, giving 2x real throughput improvement.

3% compute, 97% dispatch overhead
08

Multi-Token Kills CPU Speed

Output projections (D→vocab × 4) dominate at 8.6G FLOPs vs 1.2G for the model. Caused 2.7x slowdown and PPL 181 on CPU.

Multi-token: PPL 181, FSP: PPL 10.24

Honest Assessment

"She tried to open the key, but it was too small" — Semantic confusion. Opening a key is nonsensical; the model conflates "open" (door/lock) with "use" (key). The cumsum context maintains Lily as the actor but blends lock/key properties.

"She pulled and walked up, up the church" — Scene drift. The story shifts from park to church with no transition. The continuous cumsum state gradually drifts, losing the original setting.

Coherence breaks down ~100 tokens in. Named characters and pronoun tracking work early ("Lily... She"), but the cumsum state saturates with mixed contexts, producing word salad. Sparse memory (v9.7) improves PPL but doesn't solve this fundamental context length limitation.
